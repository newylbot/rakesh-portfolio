# Rakesh Kumar Behera — Portfolio

A premium, engineering-focused portfolio built with **Next.js (App Router) + TypeScript + Tailwind CSS**. The visual system draws from electrical-engineering language (circuit traces, oscilloscope-style waveforms, ladder-logic timelines, instrumentation labels) while staying calm, editorial, and accessible.

> **Truthfulness first.** Content is intentionally honest. Unknown links are left blank and unknown facts use the placeholder `[Confirm / add details]`. Nothing about metrics, users, production deployment, certifications, or employers is invented. The **Lumino XP** project is clearly marked **In progress**.

---

## Tech stack

- Next.js 14 (App Router)
- TypeScript
- Tailwind CSS (design tokens as CSS variables, light + dark themes)
- `next-themes` (system preference + manual toggle + persistence)
- `lucide-react` icons
- CSS/SVG-based motion only (respects `prefers-reduced-motion`) — no heavy animation library

## Getting started (run locally)

```bash
# 1. Install dependencies
npm install

# 2. Create your env file
cp .env.example .env.local
#    then set NEXT_PUBLIC_SITE_URL (and optionally CONTACT_FORM_ENDPOINT)

# 3. Start the dev server
npm run dev
#    open http://localhost:3000

# Production build
npm run build
npm run start
```

Requires Node.js 18.17+ (Node 20+ recommended).

## Editing content (no component edits needed)

All site content lives in **one file**:

```
content/site-config.ts
```

Edit it to update the name, role, headline, bio, navigation, social links, opportunities, experience, education, projects, skills, the Now log, and SEO defaults. Components read from this file, so you should almost never need to touch component code.

The Lumino XP case study has its own structured content file:

```
content/case-studies/lumino-xp.ts
```

**Rules baked into the content:**

- Leave links as empty strings (`""`) until they are real — the UI shows a clean placeholder instead of a fake button.
- Use `[Confirm / add details]` for unknown facts.
- Do not convert placeholders into real claims until confirmed.

## Design system

Color tokens are defined in `app/globals.css` as space-separated RGB channels (so Tailwind opacity modifiers like `bg-primary/20` work) and mapped to Tailwind color names in `tailwind.config.ts`. Light theme is default; `.dark` overrides. Typography uses `clamp()` scales (`display-xl`, `h1`, `body-lg`, `meta`, etc.).

To tweak the palette, edit the `:root` and `.dark` blocks in `app/globals.css`.

## Accessibility

- Semantic HTML, skip-to-content link, visible focus rings, full keyboard navigation
- Mobile menu closes on route change / `Escape` and locks body scroll
- Color is never the only status cue (icons + text + dots)
- Decorative SVGs are hidden from assistive tech; meaningful ones are labeled
- `prefers-reduced-motion` is respected globally
- Relative units (`rem` / `clamp`), zoom is never disabled

## SEO

- Per-page titles + descriptions via `lib/seo.ts`
- Canonical URLs, Open Graph, and Twitter cards
- `robots.txt` and `sitemap.xml` generated by Next.js
- Structured data for `Person` and `WebSite`

Set `NEXT_PUBLIC_SITE_URL` so canonical URLs, the sitemap, and OG tags use your real domain.

## Contact form

The form posts to `app/api/contact/route.ts`, which:

- **forwards** the submission to `CONTACT_FORM_ENDPOINT` when that env var is set, or
- returns **501** when it is not — the UI then asks visitors to email directly (no fake success).

To enable real submissions, set `CONTACT_FORM_ENDPOINT` to a real backend (e.g. Formspree, a Resend proxy, or your own API) and add a real `email` in `content/site-config.ts`.

## Deploy to Vercel

1. Push this repo to GitHub (already done).
2. Go to [vercel.com/new](https://vercel.com/new) and **Import** the `rakesh-portfolio` repository.
3. Framework preset is detected automatically (Next.js). No build changes needed.
4. Add environment variables in **Project Settings → Environment Variables**:
   - `NEXT_PUBLIC_SITE_URL` = your production URL (e.g. `https://rakesh.dev`)
   - `CONTACT_FORM_ENDPOINT` = optional, only if you have a real backend
5. **Deploy**. Add a custom domain when ready.

Build command `next build` and output are handled by Vercel's Next.js preset automatically.
